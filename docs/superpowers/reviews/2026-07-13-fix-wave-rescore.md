## Regressions introduced by the wave (actively hunted, 42 files)

1. **Dual toast hosts per screen (minor).** `page.jsx:543` renders a shell-level `toastEl` while Dashboard/HistoryView/WorkoutView/InsightsView each render their own — a shell toast (e.g. "Couldn't update log" from EditFoodModal) and a screen toast occupy the same `fixed bottom-24 z-50` slot and would overlap if concurrent. Commit `b00a07a` says "single toast host per screen"; it's actually shell + screen. Low likelihood, low harm (both `role="status"`).
2. **Edit-Meals-modal Undo asymmetry (minor).** `HistoryView.jsx:319–325` removes the row from `editingDay` before the toast resolves; Undo restores the main list but the open modal still shows the row gone (and "No entries left in this session" if it was the last). New behavior introduced by the F13 change.
3. **Not regressions, verified:** jest 39/39, production build clean, eslint 0 errors (7 pre-existing warnings), no restricted-syntax hits, manifests byte-identical after live pass, `modalStack` registry correctly handles PlateCalculator-over-ConfirmModal stacking, water rollback guards out-of-order failures, `useToast` unmount flush prevents lost deletes on tab switch.
4. **Enforcement nit (carried from F15):** the new lint rule can't see template-literal classNames, so future drift inside `` `...${x} bg-indigo-600` `` strings would pass silently. Suggest adding a `TemplateElement` selector variant.
